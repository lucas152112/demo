## 🔧 技術決策記錄

### 技術選型決策

| 技術類別 | 選擇 | 理由 | 決策日期 |
|----------|------|------|----------|
| **後端語言** | Go + Gin | 高效能、微服務友好、團隊熟悉度 | 2025-11-16 |
| **前端框架** | Vue 3 + Nuxt 3 | 現代化、SSR支援、TypeScript友好 | 2025-11-16 |
| **資料庫** | MySQL 8.0+ | 穩定可靠、ACID特性、生態成熟 | 2025-11-16 |
| **快取系統** | Redis | 高效能、豐富資料結構、session管理 | 2025-11-16 |
| **訊息佇列** | NATS | 輕量級、高效能、微服務通訊 | 2025-11-16 |
| **容器化** | Docker + Docker Compose | 環境一致性、部署便利性 | 2025-11-16 |
| **API文件** | OpenAPI 3.0 | 標準化、自動生成、前後端契約 | 2025-11-16 |

### 架構決策

| 決策項目 | 選擇 | 理由 | 決策日期 |
|----------|------|------|----------|
| **架構模式** | 微服務架構 | 可擴展性、技術多樣性、團隊自主性 | 2025-11-16 |
| **服務數量** | 8個微服務 | 職責分離、業務對應、維護性平衡 | 2025-11-16 |
| **資料庫策略** | Database per Service | 服務獨立性、避免資料耦合 | 2025-11-16 |
| **通訊模式** | RESTful API + 事件驅動 | 標準化、解耦合、非同步處理 | 2025-11-16 |
| **認證方式** | JWT + Role-Based Access Control | 無狀態、可擴展、細緻權限控制 | 2025-11-16 |
