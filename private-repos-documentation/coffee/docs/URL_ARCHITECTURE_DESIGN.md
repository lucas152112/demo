# 🌐 Coffee CRM 多權限URL架構設計

## 📋 **設計原則**
- **不同權限 → 不同登入頁面**
- **清晰的URL路徑** → 用戶角色直觀識別
- **安全隔離** → 避免權限混淆和越權訪問
- **用戶體驗** → 符合角色習慣的界面設計

---

## 🎯 **URL架構規劃**

### **🏠 主域名**: `coffee.justit.cc`

### **📱 1. 前端客戶界面** 
- **URL**: `coffee.justit.cc`
- **用途**: 客戶查看菜單、線上訂購、會員中心
- **目標用戶**: 咖啡店顧客
- **功能**: 
  - 菜單瀏覽
  - 線上訂購
  - 會員登入/註冊
  - 訂單查詢
  - 積分查看

### **🏪 2. 店員POS系統**
- **URL**: `coffee.justit.cc/pos`
- **用途**: 店員日常操作的POS收銀系統
- **目標用戶**: 咖啡店店員
- **功能**:
  - 快速收銀
  - 商品掃描
  - 會員積分
  - 基礎庫存查看
  - 交班報表

### **🏢 3. 店長管理後台**
- **URL**: `coffee.justit.cc/manage`
- **用途**: 單店管理和營運分析
- **目標用戶**: 咖啡店店長、副店長
- **功能**:
  - 員工管理
  - 庫存管理
  - 客戶管理
  - 門店報表
  - 營運分析
  - 財務管理

### **🏢 4. 總部ERP系統**
- **URL**: `coffee.justit.cc/admin`
- **用途**: 多門店管理和企業級功能
- **目標用戶**: 連鎖總部管理人員
- **功能**:
  - 多門店管理
  - 供應鏈管理
  - 財務統計
  - 品牌管理
  - 數據分析
  - 系統配置

### **⚙️ 5. 運營管理系統**
- **URL**: `coffee.justit.cc/platform`
- **用途**: 平台運營和租戶管理
- **目標用戶**: 平台營運人員 (內部)
- **功能**:
  - 租戶管理
  - 系統監控
  - 資源配置
  - 平台運營
  - 系統維護

## 🔐 **權限矩陣設計**

| URL路徑 | 用戶角色 | 權限等級 | 主要功能 | 登入方式 |
|---------|----------|----------|----------|----------|
| `/` | 顧客 | **Public** | 瀏覽、訂購 | 手機號/會員卡 |
| `/pos` | 店員 | **Staff** | POS收銀 | 員工ID/密碼 |
| `/manage` | 店長 | **Manager** | 門店管理 | 管理帳號/密碼 |
| `/admin` | 總部 | **Admin** | 企業管理 | 企業帳號/2FA |
| `/platform` | 營運 | **Platform** | 系統管理 | 內部SSO |

## 🎨 **界面設計差異**

### **🎯 用戶界面風格**
- **顧客端** (`/`): 溫馨、簡潔、品牌化
- **店員端** (`/pos`): 高效、大按鈕、快速操作
- **店長端** (`/manage`): 專業、數據化、管理導向  
- **總部端** (`/admin`): 企業級、分析導向、多維度
- **平台端** (`/platform`): 技術化、監控導向、系統級

### **🔑 認證方式差異**
- **顧客**: 手機號驗證碼 / 微信登入
- **店員**: 員工編號 + 簡單密碼
- **店長**: 用戶名 + 複雜密碼  
- **總部**: 用戶名 + 密碼 + 二次驗證
- **平台**: SSO / LDAP 企業認證

## 🚀 **實現計劃**

### **Phase 1: 路由重構** (本週)
1. **創建不同的登入頁面**
   - `/pages/index.vue` → 顧客端
   - `/pages/pos/index.vue` → 店員登入
   - `/pages/manage/index.vue` → 店長登入  
   - `/pages/admin/index.vue` → 總部登入
   - `/pages/platform/index.vue` → 平台登入

2. **設置路由保護中間件**
   - 權限驗證
   - 角色跳轉
   - 未授權攔截

3. **後端API適配**
   - 不同登入端點
   - 權限級別驗證
   - JWT角色聲明

### **Phase 2: 界面開發** (下週)
1. **設計不同的UI風格**
2. **實現角色專屬功能**  
3. **測試權限隔離**

### **Phase 3: 部署配置** (第三週)
1. **Nginx路由配置**
2. **SSL證書配置**
3. **域名解析設置**

## 📝 **技術實現要點**

### **前端路由配置** (Nuxt.js)
```typescript
// nuxt.config.ts
export default defineNuxtConfig({
  ssr: true,
  router: {
    middleware: ['auth', 'rbac']
  },
  runtimeConfig: {
    public: {
      apiBase: 'https://coffee.justit.cc:8002'
    }
  }
})
```

### **後端認證端點**
```rust
// 不同登入端點
POST /auth/customer/login    // 顧客登入
POST /auth/staff/login       // 店員登入  
POST /auth/manager/login     // 店長登入
POST /auth/admin/login       // 總部登入
POST /auth/platform/login    // 平台登入
```

### **JWT角色聲明**
```json
{
  "sub": "user_id",
  "role": "manager",
  "tenant_id": "coffee_shop_001", 
  "permissions": ["read:customers", "write:inventory"],
  "scope": "manage"
}
```
