# 需求分析文件
（測試）
## 1. 專案背景

### 1.1 專案目標
開發一套完整的咖啡館連鎖管理系統，用於支援多家連鎖門店的日常營運管理，包括：
- 門店端 POS 系統與客戶管理 (CRM)
- 總部端進銷存管理與經營分析 (ERP)
- 會員忠誠度計畫
- 營運報表與數據分析

### 1.2 目標用戶
1. **門店店員**：使用 POS 系統進行點單、收銀
2. **門店店長**：管理門店客戶、查看門店報表
3. **總部採購**：管理商品、供應商、進貨
4. **總部管理層**：查看全系統報表、分析經營數據
5. **系統管理員**：管理用戶權限、系統設定

### 1.3 業務範圍
- 支援 10-50 家連鎖門店
- 單日訂單量：每店 200-500 筆
- 商品 SKU 數量：100-500 個
- 活躍會員數：5,000-50,000 人

## 2. 功能需求 (Functional Requirements)

### 2.1 門店管理系統 (CRM)

#### 2.1.1 POS 點餐系統
**FR-CRM-001：商品選擇與下單**
- 系統應提供商品目錄瀏覽功能
- 支援商品分類篩選（咖啡、茶飲、輕食等）
- 支援商品規格選擇（冰/熱、大/中/小杯等）
- 支援加料選項（加奶、加糖、加配料等）
- 支援購物車編輯（新增、刪除、修改數量）
- 優先級：P0（核心功能）

**FR-CRM-002：訂單結帳**
- 系統應計算訂單總金額（含稅）
- 支援多種付款方式（現金、信用卡、行動支付）
- 支援會員折扣自動計算
- 支援發票列印
- 支援訂單編號生成與追蹤
- 優先級：P0

**FR-CRM-003：會員綁定**
- 系統應支援訂單綁定會員
- 支援手機號碼查詢會員
- 支援掃碼識別會員
- 優先級：P1

#### 2.1.2 客戶管理
**FR-CRM-004：客戶資料管理**
- 系統應提供客戶資料新增功能
- 客戶資料欄位：姓名、手機、Email、生日
- 支援客戶資料編輯與刪除
- 支援客戶資料搜尋與篩選
- 優先級：P0

**FR-CRM-005：客戶標籤管理**
- 系統應支援客戶標籤功能
- 支援自訂標籤（VIP、常客、潛在客戶等）
- 支援多標籤分類
- 優先級：P2

**FR-CRM-006：客戶消費歷史**
- 系統應記錄客戶完整消費歷史
- 顯示消費金額、日期、商品明細
- 支援消費記錄查詢與匯出
- 優先級：P1

#### 2.1.3 會員忠誠度
**FR-CRM-007：點數累積**
- 系統應依消費金額自動累積點數
- 點數累積比例可設定（預設：消費 $1 = 1 點）
- 支援活動加倍點數
- 優先級：P1

**FR-CRM-008：點數兌換**
- 系統應支援點數兌換商品
- 支援點數折抵金額
- 點數異動記錄完整追蹤
- 優先級：P1

**FR-CRM-009：會員等級制度**
- 系統應根據累計消費自動升級會員等級
- 不同等級享有不同折扣與優惠
- 會員等級：一般、銀卡、金卡、白金卡
- 優先級：P2

---

### 2.2 總部管理系統 (ERP)

#### 2.2.1 商品管理
**FR-ERP-001：商品資料管理**
- 系統應提供商品主檔管理
- 商品資料欄位：SKU、名稱、分類、價格、成本、圖片
- 支援商品上下架管理
- 支援商品資料匯入/匯出
- 優先級：P0

**FR-ERP-002：商品分類管理**
- 系統應支援多層級商品分類
- 支援分類新增、編輯、刪除
- 支援拖曳排序
- 優先級：P1

**FR-ERP-003：商品規格管理**
- 系統應支援商品多規格設定
- 規格類型：尺寸、溫度、甜度、配料等
- 不同規格可設定不同價格
- 優先級：P1

#### 2.2.2 庫存管理
**FR-ERP-004：庫存查詢**
- 系統應提供即時庫存查詢
- 支援多門店庫存總覽
- 支援庫存預警設定
- 庫存不足自動通知
- 優先級：P0

**FR-ERP-005：庫存盤點**
- 系統應支援庫存盤點功能
- 支援盤點單建立與審核
- 自動計算盤盈/盤虧
- 支援盤點歷史記錄
- 優先級：P1

**FR-ERP-006：庫存調撥**
- 系統應支援門店間庫存調撥
- 調撥單狀態追蹤
- 支援調撥審核流程
- 優先級：P2

#### 2.2.3 進貨管理
**FR-ERP-007：供應商管理**
- 系統應提供供應商資料管理
- 供應商資料：名稱、聯絡人、電話、地址
- 支援供應商評級
- 優先級：P1

**FR-ERP-008：採購單管理**
- 系統應支援採購單建立
- 採購單欄位：供應商、商品、數量、單價、交貨日期
- 支援採購單審核流程
- 採購單狀態：待審核、已審核、進貨中、已完成、已取消
- 優先級：P0

**FR-ERP-009：進貨入庫**
- 系統應支援進貨入庫登記
- 自動更新庫存數量
- 支援部分入庫
- 記錄進貨批次與日期
- 優先級：P0

#### 2.2.4 報表分析
**FR-ERP-010：營收報表**
- 系統應提供營收報表
- 報表維度：日報、週報、月報、年報
- 支援門店比較分析
- 支援時段分析
- 優先級：P0

**FR-ERP-011：商品銷售分析**
- 系統應提供商品銷售排行
- 顯示熱銷商品與滯銷商品
- 支援商品組合分析
- 優先級：P1

**FR-ERP-012：客戶分析**
- 系統應提供客戶統計報表
- 新增客戶數、活躍客戶數、流失客戶數
- 客戶消費習慣分析
- RFM 分析（最近、頻率、金額）
- 優先級：P1

**FR-ERP-013：毛利分析**
- 系統應提供毛利率分析
- 商品毛利率、門店毛利率
- 成本分析與趨勢
- 優先級：P2

### 2.3 系統管理

#### 2.3.1 用戶與權限
**FR-SYS-001：用戶管理**
- 系統應提供用戶帳號管理
- 支援用戶註冊、停用、刪除
- 用戶資料：帳號、姓名、角色、所屬門店
- 優先級：P0

**FR-SYS-002：角色權限管理**
- 系統應支援角色定義
- 角色類型：系統管理員、總部管理、店長、店員
- 支援細粒度權限控制（功能級別）
- 優先級：P0

**FR-SYS-003：登入與驗證**
- 系統應提供安全登入機制
- 支援 JWT Token 驗證
- 支援密碼強度檢查
- 支援登入失敗鎖定
- 優先級：P0

#### 2.3.2 門店管理
**FR-SYS-004：門店資料管理**
- 系統應提供門店主檔管理
- 門店資料：代碼、名稱、地址、電話、營業時間
- 支援門店啟用/停用
- 優先級：P0

**FR-SYS-005：門店設定**
- 系統應支援門店個別設定
- 設定項目：稅率、發票前綴、營業時間等
- 優先級：P1

#### 2.3.3 通知系統
**FR-SYS-006：郵件通知**
- 系統應支援 Email 通知
- 通知場景：訂單確認、會員註冊、密碼重設
- 優先級：P1

**FR-SYS-007：系統通知**
- 系統應支援站內通知
- 庫存預警、審核通知、系統公告
- 優先級：P2

## 3. 非功能需求 (Non-Functional Requirements)

### 3.1 效能需求
**NFR-001：響應時間**
- API 響應時間應在 200ms 以內（P95）
- POS 下單流程應在 3 秒內完成
- 報表生成應在 5 秒內完成

**NFR-002：並發處理**
- 系統應支援至少 100 個並發用戶
- 單一門店應支援至少 5 個 POS 終端同時操作

**NFR-003：資料處理量**
- 系統應支援單日 25,000 筆訂單（50 店 × 500 筆）
- 歷史資料保留至少 3 年

### 3.2 可用性需求
**NFR-004：系統可用性**
- 系統可用性應達到 99.5% (SLA)
- 計劃性維護應在非營業時段進行

**NFR-005：容錯能力**
- 單一微服務故障不應影響其他服務
- 資料庫應有自動備份機制
- 支援斷線續傳

### 3.3 安全性需求
**NFR-006：身份驗證**
- 所有 API 必須經過身份驗證
- 密碼應以 bcrypt 加密存儲
- JWT Token 有效期為 24 小時

**NFR-007：資料安全**
- 敏感資料應加密傳輸（HTTPS）
- 客戶個資應遵循 GDPR 規範
- 支援資料存取日誌記錄

**NFR-008：權限控制**
- 實施最小權限原則
- 敏感操作需要二次驗證
- 支援 IP 白名單

### 3.4 可維護性需求
**NFR-009：程式碼品質**
- 程式碼覆蓋率應達到 80% 以上
- 遵循 Go 與 Vue 最佳實踐
- 統一的錯誤處理機制

**NFR-010：文檔完整性**
- API 文檔使用 OpenAPI 3.0 規範
- 每個微服務應有 README
- 資料庫變更應有 migration 腳本

**NFR-011：監控與日誌**
- 所有服務應輸出結構化日誌
- 支援分散式追蹤（Distributed Tracing）
- 關鍵指標應有監控告警

### 3.5 擴展性需求
**NFR-012：水平擴展**
- 微服務應支援水平擴展
- 資料庫應支援讀寫分離
- 支援 CDN 加速靜態資源

**NFR-013：模組化設計**
- 前後端完全分離
- 微服務間低耦合
- 支援功能模組獨立部署

### 3.6 相容性需求
**NFR-014：瀏覽器支援**
- Chrome 最新版本
- Firefox 最新版本
- Safari 最新版本
- Edge 最新版本

**NFR-015：裝置支援**
- 桌面瀏覽器（1920x1080）
- 平板裝置（1024x768）
- 手機瀏覽器（響應式設計）

## 4. 約束條件 (Constraints)

### 4.1 技術約束
- 後端必須使用 Go 語言
- 前端必須使用 Vue 3 + Nuxt 3
- 資料庫必須使用 PostgreSQL
- 必須支援 Docker 容器化部署

### 4.2 業務約束
- 系統必須符合當地稅務法規
- 必須支援多幣別（初期僅支援 TWD）
- 發票格式必須符合電子發票規範

### 4.3 時程約束
- 第一階段（MVP）：3 個月
- 第二階段（完整功能）：6 個月
- 第三階段（優化與擴展）：持續進行

### 4.4 預算約束
- 開發團隊：5-8 人
- 伺服器預算：月費 $5,000 以內
- 第三方服務費用：月費 $2,000 以內
