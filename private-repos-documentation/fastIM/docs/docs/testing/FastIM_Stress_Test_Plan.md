# FastIM WebSocket 壓力測試計畫

## 📋 測試概覽

**測試目標**: 驗證 FastIM WebSocket 服務在高併發情況下的穩定性和性能表現  
**測試類型**: 負載測試、壓力測試、穩定性測試  
**執行環境**: Linux 生產環境  
**預期執行時間**: 30-60 分鐘  

## 🎯 測試目標與範圍

### 主要測試目標
1. **併發連接能力**: 測試最大支援的同時連接數
2. **消息傳輸性能**: 驗證高負載下的消息延遲和吞吐量
3. **系統穩定性**: 確保長時間運行下的服務穩定性
4. **資源使用情況**: 監控 CPU、內存、網絡資源消耗
5. **錯誤處理**: 驗證異常情況下的系統行為

### 測試範圍
- ✅ WebSocket 連接建立和維護
- ✅ 即時消息傳輸
- ✅ 用戶加入/離開處理
- ✅ 心跳機制穩定性
- ✅ 服務器資源監控
- ❌ 前端 UI 響應 (不在此次測試範圍)
- ❌ 數據庫性能 (獨立測試)

## 🔧 測試環境與工具

### 測試環境要求
- **操作系統**: Linux (Ubuntu 20.04+)
- **Python 版本**: 3.8+
- **網絡環境**: 穩定的網絡連接
- **系統資源**: 最少 4GB RAM, 2 CPU 核心

### 測試工具與依賴
```bash
# 安裝測試依賴
pip install websockets aiohttp asyncio psutil

# 測試工具文件
- fastim_stress_test_runner.py    # 主測試執行器
- submit_fastim_stress_test.py    # 測試提交工具
- monitor_fastim_test.py          # 資源監控工具
```

### 目標服務配置
- **測試端點**: `ws://chat.justit.cc/ws` (需確認正確路徑)
- **備用端點**: `wss://chat.justit.cc/ws` (HTTPS)
- **服務端口**: 80/443
- **協議**: WebSocket RFC 6455

## 📋 測試執行流程

### 準備階段 (5分鐘)
1. **環境檢查**
   ```bash
   # 確認網絡連通性
   curl -I http://chat.justit.cc
   
   # 檢查 WebSocket 端點
   wscat -c ws://chat.justit.cc/ws
   ```

2. **工具準備**
   ```bash
   # 下載測試工具
   cd /root/.openclaw/workspace
   python fastim_stress_test_runner.py --check
   ```

3. **基礎功能驗證**
   - 手動連接測試
   - 消息收發測試
   - 認證機制確認

### 執行階段 (45分鐘)

#### Step 1: 啟動監控
```bash
# 啟動系統資源監控
python monitor_fastim_test.py --start
```

#### Step 2: 執行測試階段
```bash
# 階段 1: 基礎測試
python fastim_stress_test_runner.py \
  --connections 50,100,150,200 \
  --duration 300 \
  --phase basic

# 階段 2: 中等負載
python fastim_stress_test_runner.py \
  --connections 200,300,400,500 \
  --duration 600 \
  --phase medium

# 階段 3: 高負載
python fastim_stress_test_runner.py \
  --connections 500,750,1000 \
  --duration 900 \
  --phase stress

# 階段 4: 穩定性測試
python fastim_stress_test_runner.py \
  --connections 300 \
  --duration 1800 \
  --phase stability
```

#### Step 3: 數據收集
```bash
# 停止監控並收集數據
python monitor_fastim_test.py --stop --report
```

### 結果分析階段 (10分鐘)
1. **日誌分析**: 檢查錯誤和異常
2. **性能指標**: 統計連接成功率、延遲分布
3. **資源使用**: 分析 CPU、內存、網絡使用情況
4. **瓶頸識別**: 確定系統性能瓶頸

## 🚨 異常處理與風險控制

### 測試過程監控
1. **實時監控**: CPU、內存使用情況
2. **連接監控**: 異常掉線、連接失敗統計
3. **服務監控**: 目標服務狀態檢查

### 異常情況處理
| 異常類型 | 處理方式 |
|----------|----------|
| 大量連接失敗 | 降低並發數，檢查端點配置 |
| 服務器無響應 | 立即停止測試，檢查服務狀態 |
| 內存使用過高 | 監控是否內存洩漏，記錄詳細信息 |
| 網絡異常 | 檢查網絡連通性，重新執行 |

### 風險控制措施
- **測試隔離**: 使用獨立測試環境
- **逐步增加**: 循序漸進增加負載
- **實時監控**: 持續監控服務狀態
- **快速停止**: 異常情況下快速終止測試
